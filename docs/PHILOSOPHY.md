# AIAL 语言设计哲学手册

## 序言
AIAL 是世界上第一个将“AI 调用”作为一等公民的原生编程语言。其使命是让开发者以最直接的路径描述 AI Agent 的意图与边界，并将描述安全、高效地转化为单文件可执行程序。

## 五大支柱
1. **零摩擦环境**：秒级编译，单二进制无依赖。
2. **原生 AI 调用**：`ask` 关键字支持流式、重试、降级、并行、Token 计费。
3. **内置上下文管理**：`context` 类型自动管理会话历史、Token 预算、DAG 摘要与检索。
4. **零成本工具暴露**：`#[tool]` 将普通函数变为 AI 工具，生成 JSON Schema，沙箱权限。
5. **渐进类型与内存安全**：开发体验如 TypeScript，编译器通过逃逸分析保证内存安全，无需生命周期标注。

## 设计原则

### 道（整体导向）
- **少则得，多则惑**：核心关键字控制在 20 个以内，同一语义只提供一种表达方式。
- **为道日损**：复杂性由编译器承担，不转移给开发者。

### 法（安全与权限）
- **法者，编著之图籍**：安全规则公开透明，`[capabilities]` 是程序契约，编译器与运行时联合执法。
- **术藏于胸中**：内存管理、所有权分析完全封装，开发者无需了解。
- **势者，胜众之资**：`context` 携带不可伪造的会话状态，动态影响工具权限。

### 禅（直指本心）
- **不立文字**：`ask` 是表达式，意图即代码。
- **直指人心**：类型推导覆盖 90% 场景，AI 返回的松散数据通过模式匹配自然归正。

### 医（可观测与资源自知）
- **望闻问切**：每次 AI 调用均内置遥测数据。
- **治未病**：提供预算预警回调，由开发者决定策略（压缩、切换模型等），语言不自动丢弃上下文。

### 兵（异常与并发）
- **以正合，以奇胜**：成功与降级/错误分支平等通过模式匹配处理，无异常语法。
- **不战而屈人之兵**：消息传递并发模型，无共享状态，无数据竞争。

### 其他映射
- **儒家**：类型即“正名”，模块秩序严格。
- **墨家**：工具接口统一契约，默认无能力（“非攻”）。

## 语言核心理念
**“人法地，地法天，天法道，道法自然。”** 开发者写下意图，编译器保证执行，AI 提供智慧，运行时承载一切复杂性。
